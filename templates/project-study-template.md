# 项目学习模板

> 说明：本模板用于**个人学习记录**，默认中文撰写。

## 1. 项目定位

- 这个项目在系统中的角色是什么？
- 它解决的核心问题是什么？
- **基准版本（pin commit）：** `<repo>@<40-char-sha>`（写清日期与选取理由）

## 2. 系统关系图

- 上游依赖
- 下游使用方
- 外部系统边界

## 3. 核心子系统清单

| 子系统 | 职责 | 关键文件 | 风险点 |
|---|---|---|---|

## 4. 关键主链路

- 请求入口 -> 核心调度 -> 状态持久化 -> 输出

## 5. 架构评价

- 设计优点（至少 3 条）
- 技术债/风险（至少 3 条）
- 如果重做，最先改哪里

## 6. 掌握验收

- [ ] 能画主链路
- [ ] 能解释边界与不变量
- [ ] 能独立定位一个中等复杂 bug 的模块归属
- [ ] 能说明“哪些设计值得复用、哪些不值得”
